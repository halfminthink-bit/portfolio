TikTok 運用（フォロワー約2,300）

背景・目的：

自分なりの分析でどこまで伸ばせるかを検証したかった。

アプローチ：

  • 同ジャンルの人気投稿を国内外でリストアップし、構成・導入・長さを分解。
  • 検証パターンを変えながら継続投稿し、数値の良い型を標準化。

LINE × YouTube要約ミニアプリ

背景・目的：

既存のWebアプリを開くのが面倒で、もっと気楽に要約したかった。

内容：

  • LINEにURLを送るだけで要約を返す仕組みを、ひとりでゼロから開発。
  • 字幕取得 → 前処理 → 要約 → 返答までを自動化し、学習効率を改善。

スキル

  • 言語：Python ／ JavaScript ／ HTML ／ CSS
  • データ：スプレッドシート／Excel（関数・ピボット・軽い可視化）
  • 自動化：スクレイピング／API連携／スケジューリング

進め方

  1. 現状把握
  2. 提案
  3. 改善
  4. 実装
  5. 評価
  6. 改善（反復）

WCM（Will / Can / Must）

  • Will： 人の時間を増やす“役に立つ仕組み”を作る。
  • Can： 小さく作ってすぐ運用に載せ、数値で検証して改善する。
  • Must： 確実・速く・透明に。報連相とログを徹底する。
